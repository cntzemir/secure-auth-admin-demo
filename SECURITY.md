# Security Policy

## Scope

This repository is an educational security-focused demo project created for learning, portfolio review, and technical discussion.

It is **not intended for production deployment** and should not be treated as a production-ready authentication system.

## Supported Versions

At the moment, only the latest version on the `main` branch is considered in scope for security-related review.

## Reporting a Vulnerability

If you notice a security issue or a weakness in the implementation, please **do not open a public GitHub issue describing exploit details**.

Instead, report it privately by email:

**canitezemirr@gmail.com**

Please include:
- a short description of the issue
- affected route, file, or feature
- reproduction steps
- potential impact
- optional mitigation suggestions

## Response Expectations

This project is maintained as a student portfolio and educational demo, so response times may vary.

Reasonable reports will be reviewed and acknowledged as soon as possible.

## Disclosure Preference

Please prefer **responsible disclosure**:
- avoid posting sensitive exploit details publicly before review
- avoid sharing secrets, personal data, or destructive payloads
- keep reports limited to safe proof-of-concept detail

## Notes

Because this is a demo project:
- some controls are intentionally simplified
- no guarantee is made for production use
- local/demo credentials and flows must be changed before any broader use
